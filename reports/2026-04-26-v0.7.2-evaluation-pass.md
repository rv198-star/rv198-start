# v0.7.2 Evaluation Pass

This report evaluates the `feature/v0.7.2-live-fact-verification` branch after the three implementation batches. It is an internal generated-artifact review plus script-based regression evidence. It is not external blind review, real-user validation, or domain-expert validation.

## Verdict

v0.7.2 proves the live fact-verification foundation and, after use-state arbitration plus chapter-title hygiene, shows positive value on high-temporal factual-safety cases and borrowed-value boundary control. The expanded generated-artifact evaluation still surfaces one release-quality caveat:

- Effective Requirements regenerated only `1` skill in this deterministic raw-book run, which is structurally thinner than the v0.6.2 workflow-heavy target shape.
- Shiji and Mao were missing from the first evaluation pass. After rerun, both keep high traditional three-layer scores. The first rerun exposed weak defer/refuse behavior (`66.9` and `69.8` proxy), but the use-state arbitration pass raises them to `94.4` and `95.2` with no wrong-verdict or boundary-leak failures.
- Mao chapter-title hygiene is now closed for the known false skill: `为争取千百万群众进入抗日民族统一战线而斗争` is filtered into `skill-hygiene-audit.json` and no longer appears in `bundle/skills/`.

Recommended decision: the current branch is now stronger for L3 proxy boundary evidence. Release should still carry a caveat for Effective Requirements breadth, but the previous Shiji/Mao/Poor Charlie proxy blocker and Mao chapter-title blocker are closed.

The correct comparison is historical context only: v0.7.1 reported KiU `97.0` vs reference `96.4` with `+0.5` average delta across `37` scenarios. v0.7.2 needs either the same reference pack used by v0.7.1 or a regenerated, aligned control pack before making a same-book delta claim.
