# v0.7.2 Evaluation Pass

This report evaluates the `feature/v0.7.2-live-fact-verification` branch after the three implementation batches. It is an internal generated-artifact review plus script-based regression evidence. It is not external blind review, real-user validation, or domain-expert validation.

## Verdict

v0.7.2 proves the live fact-verification foundation and shows positive value on high-temporal factual-safety cases, but the generated-artifact evaluation surfaces two release-quality risks:

- Poor Charlie proxy usage is only `80.4`, with failures concentrated in high-risk and edge prompts that expected refuse/defer but received apply.
- Effective Requirements regenerated only `1` skill in this deterministic raw-book run, which is structurally thinner than the v0.6.2 workflow-heavy target shape.

Recommended decision: merge is technically safe after review, but release should either carry explicit caveats or wait for a small quality pass on Poor Charlie proxy boundary and Effective Requirements workflow-heavy generation breadth.

## Same-Book Reference Benchmark

The v0.7.2 same-book reference benchmark is not currently decision-grade.

Attempts:

- Direct `.references/cangjie-skill` benchmark returned `reference_skill_count=0`, because the checkout is methodology/reference material, not a Poor Charlie skill pack in KiU bundle layout.
- A benchmark-only cangjie protocol baseline built from `bundles/poor-charlies-almanack-v0.1/sources` produced `reference_skill_count=1` and `matched_pair_count=0`, so it cannot compare the v0.7.2 Poor Charlie output against the v0.7.1 same-book pairs.

Conclusion: v0.7.2 cannot honestly claim same-book cangjie improvement from this run. The correct comparison is historical context only: v0.7.1 reported KiU `97.0` vs reference `96.4` with `+0.5` average delta across `37` scenarios. v0.7.2 needs either the same reference pack used by v0.7.1 or a regenerated, aligned control pack before making a same-book delta claim.
