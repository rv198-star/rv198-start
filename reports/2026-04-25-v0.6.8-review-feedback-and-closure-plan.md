## 2.2 工单详细卡

### KIU-686 · KiU 中文 B 段 do_not_fire enum 化升级 ⭐P0

**背景**：盲评结论 C。KiU 当前的「## B — 边界 (Boundary)」段是散文式描述（如「不要把人物名望或故事相似当作机制相同」），LLM reviewer 在 should_not_trigger 场景上无法快速命中具体诱饵类型，导致 anti_misuse 系统性偏低 1 分。

**改动范围**：所有 published KiU skill 的 [SKILL.md](http://SKILL.md) 中 B 段。本期至少覆盖：

- 史记类：consequence-judgment, role-boundary（盲评直接覆盖的两个）
- 穷查理类：bias-self-audit, circle-of-competence（参照升级，验证模板可推广）

**新格式（保持中文 + 自然语言风格）**：

```markdown
## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- **不要用于纯史实查询 / 百科查询 / 出生年问题** → 对应诱饵类型：`historical_fact_lookup`
- **不要用于古文翻译或现代汉语转换** → `classical_text_translation`
- **不要用于纯人物评价或立场表态** → `pure_character_evaluation`
- **不要用于年代线整理或事件编年** → `chronology_listing`
- **不要用于摘要 / 观点总结，缺少行动决策情境** → `pure_viewpoint_summary`
- **不要用于会议纪要 / checklist 等模板生成** → `mechanical_workflow_template`
- **不要在只有单个轶事支持当前决策时给出确定性结论** → `single_anecdote_proof`
- **不要把人物名望或故事表面相似当作机制同构** → `surface_analogy_only`

### 作者在书中警告的失败模式
（保留原有自然语言段落，不动）
```

**关键设计原则**：

1. **不复制 mock 的 YAML 结构**：保持 markdown bullet + 中文叙述。
2. **每条 bullet 末尾的  `→ 类型标签`** 是新增的可机读锚点。下游脚本（KIU-691）可解析这些标签。
3. **保留「作者在书中警告的失败模式」段**——这是 KiU 的原创风格，不要砍。

**验收标准**：

- ✅ 4 个 skill 的 [SKILL.md](http://SKILL.md) 升级完成。
- ✅ 每个 skill 的 do_not_fire 至少 5 条具名条款。
- ✅ 每条 bullet 都有 `→ <类型标签>` 后缀。
- ✅ 跑一次 v0.6.7 盲评中的 should_not_trigger 8 个诱饵 + edge_case 6 个 prompt（手工检查），KiU 应能在每个 prompt 上命中至少 1 条对应条款。

### KIU-690 · Consequence skill 增加 scenario_families 段 ⭐P0

**背景**：盲评结论 B。KiU consequence skill 当前在 should_trigger 场景上 0 胜，因为 reviewer 找不到「这个 prompt 应该激活我」的明确锚点。mock 的 `scenario_families` 字段（4 个具名场景：historical-analogy-for-current-decision / suggestive-but-different-context / single-anecdote-proof / history-summary-only）在 LLM 命中信号上明显占优。

**改动范围**：consequence-judgment skill（史记 bundle）+ 任何包含「历史/案例 → 当下决策」语义的 skill。

**新增段（位置：放在 I 方法论骨架之后、E 可执行步骤之前）**：

```markdown
## S — 适用场景族 (Scenario Families)

### 正面场景：historical-analogy-for-current-decision
- **用户在说**：「能不能用 X 历史故事判断我现在的 Y 决策」「项羽和刘邦的故事能不能类比 ___」
- **判断信号**：用户已有当下决策上下文 + 想拿历史案例做后果压力测试
- **下一步**：进入 E 段四步执行，重点在机制链而不是人物类比

### 正面场景：short-gain-long-cost-stress-test
- **用户在说**：「眼前收益大但担心反噬」「短期赢一时但长期失信」
- **判断信号**：短期利益 vs 长期代价的二阶后果担忧
- **下一步**：用 E 段构建 选择 → 约束变化 → 后续报复/信任折损 链条

### 边缘场景：suggestive-but-different-context
- **用户在说**：「有点像但时代不一样」「关键角色的激励可能不同」
- **判断信号**：相似机制存在但关键变量不同
- **下一步**：先列出相似点 + 关键差异 + 不可迁移部分，再 partial_apply（不给确定性结论）

### 拒绝场景：single-anecdote-proof
- **用户在说**：「我只记得一个史记故事，想拿来支持我的方案」
- **判断信号**：单孤立轶事 + 想直接背书当前方案
- **下一步**：要求补充 case_analogs + relevant_differences；证据不足时输出 do_not_apply

### 拒绝场景：history-summary-only
- **用户在说**：纯史实查询 / 翻译 / 编年 / 人物评价
- **判断信号**：缺少 current_decision，仅查询历史本身
- **下一步**：不激活本 skill（与 B 段 do_not_fire 对齐）
```

**关键设计原则**：

1. **保持中文叙述风格**——不要写成 mock 那种 `signals: ; boundary: ; next:` YAML，而是用「用户在说 / 判断信号 / 下一步」三件套子标题，对 LLM 命中友好且对人类阅读自然。
2. **每个场景族对应 1 个 ID 标签**（用于 KIU-691 校验）。
3. **正面 / 边缘 / 拒绝 三类必须齐备**——盲评 case_type 4 类的对应物。

**验收标准**：

- ✅ consequence-judgment skill 至少 5 个场景族（2 正面 + 1 边缘 + 2 拒绝）。
- ✅ 重跑 v0.6.7 盲评中 p001-p013 的 13 个 prompt，KiU 应能在每个 prompt 上命中至少 1 个 scenario_families 条款。
- ✅ B 段 do_not_fire（来自 KIU-686）与 S 段拒绝场景族**条款互不重复**——B 段是泛规则、S 段是命中模板。
