# v0.5.1 Cangjie Gap Closure Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the specific gaps that prevent `v0.5` from being honestly described as having re-absorbed `cangjie-skill` and achieved a same-source, same-scenario win.

**Architecture:** Treat `v0.5.1` as a narrow patch train, not a new foundation rewrite. Reuse the current benchmark, three-layer review, book-to-skill path, and boundary stack, but only ship the subset that directly closes the `cangjie` comparison gap: benchmark coverage, same-scenario usage quality, book-to-skill production proof, and release narrative alignment. Any `Graphify`-style graph-model expansion stays explicitly outside this version.

**Tech Stack:** Python 3, `unittest`, existing `scripts/benchmark_reference_pack.py`, `scripts/run_book_pipeline.py`, `scripts/review_generated_run.py`, YAML/JSON reports, current Poor Charlie benchmark corpus and alignment file.

---

## Scope Definition

`v0.5.1` exists to close one mismatch:

- `v0.5 foundation complete`
- but `v0.5 cangjie-overall-win not achieved`

Version intent after this correction:

- `v0.5.x` = `cangjie-skill` absorption / benchmark closure line
- `v0.6` = `Graphify` alignment line

This mismatch is already archived in:

- [`reports/2026-04-23-v0.5-vs-cangjie-conclusion.md`](../../../reports/2026-04-23-v0.5-vs-cangjie-conclusion.md)

Current blocking evidence:

- same-scenario usage benchmark is still behind the reference line
  - KiU average usage score: `69.4`
  - reference average usage score: `72.5`
  - average delta: `-3.2`
- same-book benchmark still records open carry-over gaps around behavior quality
- the old claim “schema complete == cangjie gap closed” is unsupported

Primary evidence files:

- [`reports/w2-benchmark-summary.md`](../../../reports/w2-benchmark-summary.md)
- [`reports/2026-04-23-v0.5-vs-cangjie-conclusion.md`](../../../reports/2026-04-23-v0.5-vs-cangjie-conclusion.md)
- [`benchmarks/alignments/poor-charlies-vs-cangjie.yaml`](../../../benchmarks/alignments/poor-charlies-vs-cangjie.yaml)

## Non-Goals

`v0.5.1` does **not** include:

- broad `Graphify` absorption work
- graph-schema or provenance-contract expansion whose primary purpose is `Graphify` alignment rather than `cangjie` gap closure
- MCP/runtime/dispatcher revival
- new domain expansion
- count-only throughput chasing that weakens the `Agentic vs Workflow` boundary

## Release Gate For v0.5.1

`v0.5.1` is only valid if all items below are true on fresh evidence:

1. same-source benchmark is rerun and archived from a fresh Poor Charlie generated run
2. concept-aligned artifact review still shows KiU ahead of the reference pack
3. same-scenario usage average delta is `> 0.0`
4. same-scenario usage winner is `kiu`
5. same-scenario weighted pass rate is `>=` the reference weighted pass rate
6. workflow boundary remains preserved; no “win” comes from false workflow promotion
7. the repo narrative explicitly states:
   - `v0.5.0 = foundation complete`
   - `v0.5.1 = cangjie gap closure`

Note added on `2026-04-24`:

- the earlier strict `weighted_pass_rate > reference` rule was retired because it becomes
  impossible once both sides saturate at `1.0`
- the corrected standard is: `v0.5.1` must show slight but explicit benchmark superiority,
  not just structural parity

## Exit Rule

Do **not** release `v0.5.1` if any of these remain true:

- KiU still loses on same-scenario usage average
- KiU does not become the same-scenario usage winner
- KiU falls below the reference weighted pass rate
- benchmark gains depend on weakening the workflow-vs-agentic boundary
- the repo narrative still implies that `v0.5.0` already finished this line

If those blockers remain, continue development under `v0.5.1` until the scorecard can
prove closure.
