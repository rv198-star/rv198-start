# KiU v0.4 Workstream Decomposition Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn the v0.4 handoff report into a locally grounded execution roadmap that matches the real v0.3.0 repository state and the mandatory v0.3.1 gate fixes.

**Architecture:** Split v0.4 into five workstreams instead of treating the 13 tickets as one monolith. The first workstream is content/eval honesty on the investing bundle, the second is the engineering domain, the third is cross-bundle graph support, the fourth is real refinement execution, and the fifth is release/docs. This keeps the schedule aligned with the current codebase and avoids letting outdated assumptions leak into execution.

**Tech Stack:** Python 3, YAML bundle assets, `unittest`, existing validator/pipeline/refiner code, GitHub Actions, OpenAI or Ollama for real refinement runs.
