# v0.6 W3 Full L3 Quality Purification Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Finish the L3 quality-purification phase by making usage-quality diagnostics actionable, sharpening generated contracts, and moving release judgment from artifact-only to behavior-aware.

**Architecture:** Keep the current W2 production pipeline intact, then extend three connected layers. First, generated contracts become more scenario-specific so usage quality can improve for the right reason. Second, same-scenario usage review rolls failures up into upstream repair owners so L3 output is no longer a dead-end score. Third, three-layer review exposes a behavior-aware release gate that can block superficially clean but usage-weak runs.

**Tech Stack:** Python 3, `unittest`, existing `run_book_pipeline.py`, `review_generated_run.py`, `benchmark_reference_pack.py`, YAML/JSON artifacts.
