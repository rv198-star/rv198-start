# v0.7.1 World Alignment Value Evidence Design

## Goal

v0.7.1 closes the gap identified by the v0.7 review: v0.7.0 proves isolated world-alignment structure, but does not yet provide release-grade internal gate evidence or internal value-delta evidence.

v0.7.1 is one version with two implementation batches and two commits:

1. Evidence transparency closure.
2. Value metrics and on/off ablation.

## Claim Boundary

v0.7.1 may claim internal value signals for isolated world alignment. It may not claim external blind preference, real-user validation, live-web factual correctness, domain-expert validation, or multi-world modeling.

## Batch 1: Evidence Transparency Closure

Batch 1 produces a release-grade internal gate scorecard. It must report actual values, thresholds, pass/fail status, and evidence paths for the v0.7 charter gate:

- `source_fidelity_preserved`
- `world_context_isolated`
- `workflow_boundary_preserved`
- `samples_passed`
- `world_alignment_score_min`
- `world_context_depth_score_min`
- `source_pollution_errors`
- `original_source_only_mode_supported`
- `application_gate_cases`

Batch 1 also reports mechanism run counts for need scoring, intervention levels, candidate arbitration, no-forced-enhancement, source-fit review, dilution-risk review, and hallucination-risk review.

## Batch 2: Value Metrics And Ablation

Batch 2 adds internal value metrics that sit between structure tests and external human validation:

| Metric | Release Gate | Stretch Target |
| --- | ---: | ---: |
| `alignment_usage_delta` | `>= +0.3` | `>= +0.8` |
| `misuse_intercept_rate` | `>= 95%` | `>= 98%` |
| `temporal_sensitivity_response` | `>= 90%` | `>= 95%` |
| `verdict_diversity_score` | `>= 4 verdict types` | `5 verdict types` |
| `no_forced_enhancement_rate` | low-need samples `>= 20%` | `>= 30%` |
| `source_pollution_errors` | `0` | `0` |
| `application_gate_cases` | `>= 30` | `>= 45` |
| `samples_passed` | `>= 3` | `>= 4 including adversarial fixture` |

`alignment_usage_delta` intentionally uses a modest release gate. Proxy usage remains internal evidence and must not be optimized into a self-referential benchmark.

## Reports

Batch 1 creates `reports/2026-04-25-v0.7.1-world-alignment-gate-evidence.md`.

Batch 2 creates `reports/2026-04-25-v0.7.1-world-alignment-value-metrics.md`.

The release narrative must keep proxy/ablation evidence separate from real-world effectiveness.

## Non-Goals

- No live web retrieval.
- No external reviewer workflow.
- No mutation of source-faithful `SKILL.md` artifacts.
- No hidden ingestion of external final skill packs.
- No broad rewrite of the v0.7 world-alignment generator.
