# v0.7.2 Use-State Arbitration Design

## Goal

v0.7.2 should move world alignment from a sidecar explanation into an auditable application decision model. The target is not to make world context richer by itself. The target is to decide when source knowledge is allowed to be used, when it must ask for more context, when it must refuse, and when current facts must be verified.

## Problem Statement

The v0.7.2 evaluation pass shows a measurement split:

- Traditional three-layer scores stay high for Shiji and Mao (`95.5` and `94.5`).
- Proxy usage scores are low for Shiji and Mao (`66.9` and `69.8`).
- Failures concentrate in high-risk, missing-context, and edge prompts that expected `defer`, `ask_more_context`, or `refuse`, but received `apply`.

This means the world layer is structurally isolated and safe, but it does not yet control application behavior. A scoring tweak would be too shallow. KiU needs a use-state arbitration layer.

## Model

The application chain becomes:

```text
User Request
  -> Use-State Classifier
  -> Source Skill Candidate
  -> Evidence Sufficiency Gate
  -> World / Live Fact Gate
  -> Final Application Verdict
```

The world model becomes one evidence source in arbitration, not the owner of final behavior.

## Use States

| use_state | Meaning | Default behavior |
| --- | --- | --- |
| `source_understanding` | Summary, translation, biography, fact lookup, viewpoint explanation. | `do_not_apply` |
| `low_risk_reflection` | Low-risk reflection or principle internalization without operational consequence. | `apply_with_caveats` |
| `bounded_application` | Explicit current decision, adequate context, bounded downside. | `apply` |
| `context_insufficient` | Current decision exists but key context is missing or user asks for direct conclusion without context. | `ask_more_context` |
| `transfer_candidate` | User wants to transfer a book, case, history, analogy, or principle to a present situation. | Require transfer checks before apply. |
| `transfer_abuse_risk` | User wants to copy a source, case, authority, competitor, or big-company practice without fit checks. | `refuse` or `ask_more_context` |
| `current_fact_required` | Prompt depends on current prices, markets, laws, policies, people, data, or other time-sensitive facts. | Require fact verification or block direct apply. |

## Verdict Rules

- `source_understanding` must not fire an agentic skill.
- `current_fact_required` must not direct-apply without verified facts.
- `transfer_abuse_risk` must not direct-apply.
- `context_insufficient` must not direct-apply.
- `transfer_candidate` may only apply when mechanism mapping, transfer conditions, and anti-conditions are present.
- `low_risk_reflection` should not be over-gated by world pressure. It should preserve source richness and use only minimal or light intervention.
- `bounded_application` may apply when evidence and context are sufficient.

## Acceptance Gates

### Gate 1: Use-State Classification

- Add at least 28 deterministic fixtures across Shiji, Mao, Financial Statement, Effective Requirements, and Poor Charlie.
- Every use state has at least 3 positive cases and at least 1 negative case.
- Fixture accuracy is 100%.

### Gate 2: Final Verdict Safety

- `source_understanding` never returns `apply`.
- `context_insufficient` never returns `apply`.
- `transfer_abuse_risk` never returns `apply`.
- `current_fact_required` without verified facts never returns `apply`.
- `transfer_candidate` without required transfer evidence never returns `apply`.

### Gate 3: Proxy Usage Readiness

- Shiji proxy score >= 85 and `gate_ready=true`.
- Mao proxy score >= 85 and `gate_ready=true`.
- Poor Charlie proxy score >= 85 and `gate_ready=true`.
- Financial Statement proxy score remains >= 95.
- Effective Requirements proxy score remains >= 95.
- Five-sample average proxy score >= 90.
- `boundary_leak=0` and `wrong_verdict=0` for release readiness.

### Gate 4: Non-Dilution

- Low-risk reflection prompts are not forced into live fact checks.
- Source-only mode remains supported.
- Source `SKILL.md` files remain unmodified by world/live overlays.
- `source_pollution_errors=0`.

### Gate 5: Live Fact Regression

- Existing live fact verification gates remain passing.
- Unsupported, conflicting, stale, undated, insufficient, and retrieval-failed facts cannot direct-apply.
- Real live retrieval evidence remains separated from fixture evidence.

### Gate 6: Generation Hygiene

- Chapter-title-derived candidates can be audited but must not be published as judgment skills.
- Mao rerun must not publish the known chapter-title-style candidate as a skill.

## Scope

This is an application arbitration change, not a source extraction rewrite and not a world-context content expansion. It must preserve the external reference boundary, source-only mode, and workflow-vs-agentic boundary.
