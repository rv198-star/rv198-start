# v0.6 W3 L3-1 Usage Failure Attribution Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn same-scenario usage review into a failure-attribution layer that explains why KiU loses on usage quality, not just by how many points.

**Architecture:** Keep the current `reference_benchmark` pipeline as the single source of truth for same-scenario usage review. Extend per-case evaluation with structured failure analysis, then roll those tags up into pair-level and run-level summaries and expose them in the markdown report. Do not change generation logic in this task; this task is diagnosis-first.

**Tech Stack:** Python 3, `unittest`, existing `scripts/benchmark_reference_pack.py`, YAML/JSON benchmark artifacts.
