# v0.8.1 Repository Structure Cleanup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Reorganize the repository so current project docs, current reviewer artifacts, engineering assets, evidence, examples, and archived process material have clear, stable homes.

**Architecture:** Use `git mv` for tracked moves, keep `review-pack/current/` as the reviewer-facing current artifact entry, and introduce indexes before or during migration so active entry points stay understandable. Historical report contents are not rewritten except when updating active indexes and tests.

**Tech Stack:** Git file moves, Markdown indexes, Python `unittest`, shell structure checks, `rg` link checks.

---

## File Structure

Create:

- `docs/README.md`: top-level docs navigation.
- `docs/public/`: public/current docs.
- `docs/engineering/`: engineering docs, specs, skill specs, evaluation docs, usage guide.
- `docs/release-history/`: version plans, implementation plans, review-response docs.
- `docs/archive/`: early background and historical context.
- `evidence/README.md`: evidence navigation and evidence-boundary rules.
- `evidence/current/`: current closeout/reference evidence.
- `evidence/archive/`: historical release evidence and blind-review packs.
- `review-pack/README.md`: current-review-pack navigation.
- `bundles/README.md`: legacy/sample bundle status.
- `_archive/notion-export/`: downgraded Notion export material.

Move:

- Public docs from `docs/*.md` to `docs/public/`.
- Engineering docs from `docs/*.md` and `docs/superpowers/specs/` to `docs/engineering/`.
- Plans and release/history docs from `docs/*.md` and `docs/superpowers/plans/` to `docs/release-history/`.
- Early history docs to `docs/archive/`.
- `reports/` contents to `evidence/current/` and `evidence/archive/`.
- `examples` source materials into `examples/source-materials/`.
- `workflow_candidates/examples/` into `examples/legacy/workflow-candidates/`.
- `notion-export/` into `_archive/notion-export/`.

Modify:

- `README.md`: update active documentation and evidence links.
- `AGENTS.md`: point to repository-structure policy and updated paths.
- `CHANGELOG.md`: record v0.8.1 repository structure cleanup.
- `tests/test_packaging.py`: update path-sensitive assertions for examples/docs if required.
- `.gitignore`: add local-only cleanup patterns if discovered during migration.
