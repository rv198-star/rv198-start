# v0.6 W2 Production Pipeline Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extend the current W1 provenance baseline into the first auditable `raw book -> extraction -> verification -> routing -> review` production path without breaking the workflow-vs-agentic boundary.

**Architecture:** Keep the existing `src/kiu_pipeline` path as the backbone instead of starting a second pipeline. Add the missing upstream context artifact, real extractor orchestration, a pre-routing verification gate, and post-merge inferred connections. Routing stays profile-owned and evidence-driven; no step may trade quality gains for boundary drift.

**Tech Stack:** Python 3, `unittest`, YAML/JSON artifacts, existing `scripts/*` CLIs, existing graph/report/benchmark stack, `../v0.5-foundation/.venv/bin/python`.
