# v0.7.3 Evaluation Taxonomy Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Reorganize KiU evaluation into method integrity, source/fact safety, action value, and evidence confidence without changing default generation behavior.

**Architecture:** v0.7.3 is a documentation and evidence-structure release. It updates AGENTS methodology, adds metric taxonomy documents, retrofits v0.7.2 evidence into the new A/B/C structure, then runs an existing generated-output evaluation to check whether the taxonomy explains real artifacts.

**Tech Stack:** Markdown evidence files, existing KiU Python scripts, existing JSON/Markdown reports, `unittest` sanity checks.
