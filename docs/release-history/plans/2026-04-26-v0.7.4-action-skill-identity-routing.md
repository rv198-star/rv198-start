# v0.7.4 Action-Skill Identity Routing Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an action-skill identity layer and semantic candidate routing so source containers do not become published `SKILL.md` artifacts just because they are source-backed and structurally complete.

**Architecture:** v0.7.4 inserts a model boundary between candidate seed mining and publication. Batch 1 adds diagnostics only. Batch 2 makes routing affect publication. Batch 3 reruns five books and validates C-class score improvements without weakening source/fact safety or workflow-vs-agentic boundaries.

**Tech Stack:** Python 3.10+, `unittest`, existing `kiu_pipeline` candidate, seed, quality, render, and review modules, JSON/Markdown reports.
