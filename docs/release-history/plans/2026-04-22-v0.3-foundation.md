# v0.3 Foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the v0.3 foundation layer by introducing domain profiles and hardening the validator around trigger registry, rationale density, published eval minimums, revision-loop requirements, and relation target consistency.

**Architecture:** Add `shared_profiles/<domain>/` as the new source of inherited profile defaults, resolve it during bundle loading and validation, and make the validator consume those resolved thresholds and registries. Keep existing v0.2 pipeline behavior working by treating bundle-local automation as overrides and by surfacing warnings separately from errors.

**Tech Stack:** Python 3.12, `unittest`, YAML-based bundle/profile schemas, existing `kiu_validator` and `kiu_pipeline` modules.
