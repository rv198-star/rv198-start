# v0.7.2 Use-State Arbitration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a use-state arbitration layer so KiU can decide when knowledge may be applied, when it must defer, and when current facts must be verified.

**Architecture:** Add a focused `kiu_pipeline.use_state` module with a deterministic classifier, evidence sufficiency gate, and final verdict composer. Integrate it into proxy usage first, then world/application gates after the behavior is proven by tests.

**Tech Stack:** Python standard library, `unittest`, YAML artifact overlays, existing KiU pipeline scripts.
