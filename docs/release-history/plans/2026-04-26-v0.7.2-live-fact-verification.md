# v0.7.2 Live Fact Verification Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add bounded live web fact verification for current-world claims while preserving source-faithful skills and no-web safety behavior.

**Architecture:** v0.7.2 adds an isolated claim-ledger and external-fact-pack layer under `world_alignment/`. Retrieval is workflow-style and bounded to specific claims; claim support verification and application-gate decisions are explicit, auditable, and prevented from mutating `SKILL.md`.

**Tech Stack:** Python 3.10+, `urllib.request`, `yaml`, `unittest`, existing KiU scripts, existing `world_alignment.py` and `world_alignment_metrics.py` patterns.

---

## File Map

- Create `src/kiu_pipeline/claim_ledger.py`: convert high-temporal/no-web hallucination signals into structured claims requiring verification.
- Create `src/kiu_pipeline/live_facts.py`: bounded live retrieval adapter and fact-pack writer; no advice generation.
- Create `src/kiu_pipeline/fact_verification.py`: claim-to-evidence relation classifier and validation helpers.
- Create `src/kiu_pipeline/freshness_gate.py`: map verification status to application-gate decisions.
- Modify `src/kiu_pipeline/world_alignment.py`: consume fact verification results without mutating source-derived skills.
- Modify `src/kiu_pipeline/world_alignment_metrics.py`: add value/regression attribution helpers for live-on/live-off evidence.

```python
CLAIM_LEDGER_SCHEMA = "kiu.claim-ledger/v0.1"

def build_claim_ledger(bundle_id: str, records: list[dict], mode: str = "no_web") -> dict:
    claims = []
    for index, record in enumerate(records, 1):
        text = str(record.get("prompt") or record.get("reason") or "")
        temporal = str(record.get("temporal_sensitivity") or "low")
        requires = temporal == "high" and _contains_current_fact_marker(text)
        if not requires:
            continue
        claims.append(
            {
                "claim_id": f"claim-{index:03d}",
                "bundle_id": bundle_id,
                "skill_id": record.get("skill_id"),
                "text": text,
                "claim_type": _classify_claim_type(text),
                "origin": record.get("origin", "application_gate"),
                "temporal_sensitivity": temporal,
                "verification_required": True,
                "no_web_status": "blocked_current_fact_claim" if mode == "no_web" else "live_check_required",
                "allowed_without_web": False,
                "verification_status": "pending",
            }
        )
    return {"schema_version": CLAIM_LEDGER_SCHEMA, "bundle_id": bundle_id, "claims": claims}
```
