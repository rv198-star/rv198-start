# v0.5 Workflow Delivery And Three-Layer Scoring Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the remaining v0.5 review gaps by turning downgraded workflow artifacts into minimum deliverable skeletons and emitting auditable 100-point scores for source bundle, generated bundle, and usage outputs.

**Architecture:** Extend the existing generated-run renderer instead of inventing a new package format. Workflow artifacts remain outside `bundle/skills/`, but now gain structured `workflow.yaml` plus a human-readable `CHECKLIST.md`. Add a lightweight review-scoring module/script that reuses validator and production-quality reports for the first two layers and scores usage-review YAML files via a fixed rubric for the third layer.

**Tech Stack:** Python 3, unittest, YAML/JSON reports, existing KiU pipeline render/quality/validator modules.
