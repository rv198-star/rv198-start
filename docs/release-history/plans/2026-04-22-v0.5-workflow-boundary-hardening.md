# v0.5 Workflow Boundary Hardening Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the workflow-vs-context/agentic boundary a hard, executable standard in v0.5 by proving the engineering second domain can generate both skill candidates and downgraded workflow artifacts from the same source bundle.

**Architecture:** Keep the routing rule in bundle profile metadata, not in ad-hoc runtime heuristics. Harden the engineering source bundle so its skill-principle nodes map back to real source skills, and add one explicit control node that must downgrade to `workflow_script_candidate` and stay outside `bundle/skills/`.

**Tech Stack:** Python 3, unittest, YAML source bundles, KiU candidate generator, KiU validator/preflight.
