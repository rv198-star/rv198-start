# KiU v0.6 Regression Baseline

## Purpose

`v0.6` starts from a constraint: the `v0.5` foundation must not regress while the
upstream raw-book production pipeline is added. This document freezes the minimum
regression baseline required by `KIU-610`.

## Fixed Checks

The baseline is defined as the following ordered checks:

1. `unit-tests`
   - `python -m unittest discover tests`
2. `validate-investing-merge`
   - `python scripts/validate_bundle.py bundles/poor-charlies-almanack-v0.1 --merge-with bundles/engineering-postmortem-v0.1`
3. `validate-engineering-merge`
   - `python scripts/validate_bundle.py bundles/engineering-postmortem-v0.1 --merge-with bundles/poor-charlies-almanack-v0.1`
4. `build-investing`
   - `python scripts/build_candidates.py --source-bundle bundles/poor-charlies-almanack-v0.1 ...`
5. `review-investing`
   - `python scripts/review_generated_run.py --run-root <generated investing run> --source-bundle bundles/poor-charlies-almanack-v0.1`
6. `build-engineering`
   - `python scripts/build_candidates.py --source-bundle bundles/engineering-postmortem-v0.1 ...`
7. `review-engineering`
   - `python scripts/review_generated_run.py --run-root <generated engineering run> --source-bundle bundles/engineering-postmortem-v0.1`

## Canonical Runner

The fixed baseline is executed by:

```bash
PYTHONPATH=src ./.venv/bin/python scripts/run_v06_regression_baseline.py
```

If the repository does not keep a local virtual environment, pass an explicit
interpreter:

```bash
PYTHONPATH=src ../v0.5-foundation/.venv/bin/python scripts/run_v06_regression_baseline.py \
  --python-executable ../v0.5-foundation/.venv/bin/python
```

The runner writes artifacts to `/tmp/kiu-local-artifacts/regression-baseline` by
default, or to `$KIU_LOCAL_OUTPUT_ROOT/regression-baseline` when that environment
variable is set.

## Output Contract

The canonical report is:

- `reports/v0.6-regression-baseline.json`

The runner also writes per-check logs:

- `logs/<check-id>.stdout.log`
- `logs/<check-id>.stderr.log`

Generated candidate runs live under:

- `generated/poor-charlies-almanack-v0.1/baseline-investing/`
- `generated/engineering-postmortem-v0.1/baseline-engineering/`

## Development Workflow

- Default mode: run the full baseline before and after W1 changes.
- Focused mode: use `--only <check-id>` to rerun a subset while developing.
- Release mode: the full fixed sequence above must pass again before claiming the
  foundation stayed intact.
