# KiU v0.4.1 汇总评估与 v0.4.2 优化计划

## 一句话结论

`v0.4.1` 的真实定位应当是“单 domain 生产线与内容质量强化版”，而不是原始定义里那个已经完成多 domain / 跨 bundle 假设验证的 `v0.4`。

## 汇总结论

外部评审和本地 real-use 复核在大方向上是一致的：

- `v0.3.1` 的两个核心漏洞修复是成立的，而且执行得比原计划更严格。
- 5 条 investing published skills 的内容厚度、关系表达和 canonical trace 绑定都明显上了一个台阶。
- `production_quality` 这套质量门禁是正向增量，说明工程面开始从“结构合法”走向“产物分级”。

但也有三件事必须说清楚：

- `v0.4.1` 没有完成原始 `v0.4` 里承诺的第二 domain、跨 bundle 联合图、`spec-v0.4` 文档闭环。
- `refinement_scheduler` 还没有在已发布 skill 上形成可审计的真实 loop 证据，因此不能把手工改稿写成 autonomous refinement。
- 真实消费复核暴露了两个产品级问题：
  - 生成后的 `SKILL.md` Evaluation Summary 可能和 `eval/summary.yaml` 不一致
  - `lock-onto-accounting-value` 在 `story-only` 对抗样本上存在 underfire 风险

## 当前事实判断

### 已完成并可信的部分

- 单 domain reference bundle 已具备厚 skill、双层锚定、共享 traces / evaluation、domain-profile validator。
- 两条 example fixture 线路已经能独立产出可评测的 generated bundles。
- 默认本地产物目录已经固定到 `/tmp/kiu-local-artifacts`，仓库纪律明显更稳。

### 还不能对外夸大的部分

- 不能说 KiU 已完成跨 domain 假设验证。
- 不能说 refinement loop 已被真实证明能把 shipped skill 自动推到更高质量。
- 不能把 `production_quality: excellent` 等同于“真实 runtime 消费已被证明”。

## v0.4.2 工单

### KIU-421 文档与版本叙事闭环

- 补 `CHANGELOG.md` 的 `0.3.1 / 0.4.0 / 0.4.1`
- 新增 `docs/kiu-skill-spec-v0.4.md`
- 在 README / usage guide 中明确 `v0.4.x = 单 domain 生产线强化线`

### KIU-422 生成摘要诚实同步

- `SKILL.md` 的 `Evaluation Summary` 必须由 `eval/summary.yaml` 反向渲染
- `SKILL.md` 的 `Revision Summary` 必须由 `iterations/revisions.yaml` 反向渲染
- refinement mutation 后必须刷新 markdown，而不是只 bump metadata
- preflight 必须能抓到 summary drift

### KIU-423 story-only 路由修复

- `lock-onto-accounting-value` 需要覆盖 `valuation_depends_only_on_story_or_long_term_guess`
- 不能再把最该触发的 adversarial symbol 放到 exclusion 里

### KIU-424 修订诚实标注

- reference investing bundle 的 revision 4 统一标记为 manual content upgrade
- `SKILL.md` 与 `revisions.yaml` 都要显式说明“不是 loop-driven”

### KIU-425 profile 继承字段收口

- runtime 接受 `inherits_from`
- 保留 `inherits` 兼容旧 bundle
- 文档明确推荐新 bundle 用 `inherits_from`

## v0.5 延后工单

- `engineering` domain profile 与独立 source/published bundle
- 跨 bundle graph merge 与 validator `--merge-with`
- 至少 2 条 skill 的真实 refinement loop 跑通并留存可审计证据

## 验收口径

`v0.4.2` 的完成标准不是“把 v0.5 偷偷做完”，而是把当前这条线讲清楚、修干净、跑一致：

- 版本叙事和仓库事实一致
- 生成厚视图不再对 YAML 真源说谎
- 已知真实消费 P0 风险被修掉
- 已发布 revision 文案不再伪装成 autonomous loop
