# KiU v0.6.4 Borrowed Value Maximization Plan

Date: `2026-04-24`

## 背景

v0.6.3 已经证明 KiU 可以从《史记》这类多文件历史素材做 raw-book no-seed cold start，并在 explicit cangjie protocol baseline 上小幅胜出。但这也暴露了一个更通用的问题：传记、历史、案例、战略论述类素材往往过于厚重，若按章节覆盖或素材覆盖生成 skill，容易产出“什么都能讲、但不知道何时使用”的泛化空洞。

v0.6.4 的目标不是为《史记》或《毛泽东选集》做专属补丁，而是定义并落地一条通用生成策略：

```text
For biography / history / case-heavy sources:
maximize transferable use value, not textual coverage.
```

中文名：**借鉴价值最大化策略**。

## 核心原则

### 1. Source Shape 驱动，不按书名特化

触发条件来自素材形态，而不是书名：

```yaml
source_shape:
  biography_heavy: optional
  history_heavy: optional
  case_heavy: optional
  narrative_case_heavy: optional
  argument_strategy_heavy: optional
  policy_boundary_heavy: optional
```

《史记》主要压测 `narrative_case_heavy`；`MaoZeDongAnthology-master` 主要压测 `argument_strategy_heavy + policy_boundary_heavy + historical_case`。

### 2. Graph 层完整抽取，Skill 层拒绝误触发

“拒绝摘要、翻译、史实查询、人物介绍、立场评论”只约束 judgment skill 的触发入口，不代表素材基座不抽取。

```yaml
graph_layer:
  must_extract:
    - summaries
    - original_text_segments
    - factual_entities
    - biographical_entities
    - positions_and_claims
    - chronology
    - source_locations
    - argument_structure
    - case_mechanisms
  purpose:
    - evidence_pool
    - retrieval_anchors
    - graph_navigation
    - auditability

skill_layer:
  must_reject_as_trigger:
    - pure_summary_request
    - pure_translation_request
    - fact_lookup_request
    - biography_intro_request
    - author_position_query
    - stance_commentary_without_user_decision

routing_layer:
  should_route_to:
    - retrieval_qa
    - summarization
    - translation
    - source_browser
    - workflow_candidate
```

### 3. 借鉴价值优先于文本覆盖

不问“这本书有哪些内容都要覆盖”，而问：

```text
这本书能在哪些现代判断场景中提供可迁移的机制、反例、边界和压力测试？
```

候选 skill 必须有：

- `modern_trigger`
- `transfer_conditions`
- `anti_conditions`
- `decision_schema`
- `abuse_or_overreach_boundary`

单人物、单事件、单案例不得直接发布为 thick judgment skill，只能进入 evidence pool 或 edge-case。

## 目标产物

v0.6.4 不引入 world-alignment；它仍属于 book/source-to-skill 生产质量层。

计划产物：

- `borrowed_value_maximization` 生成策略。
- `source_shape_classifier` 对 biography/history/case/argument-strategy 素材打形态标签。
- `case_mechanism` 与 `situation_strategy_pattern` 两类 graph 节点/关系。
- `transfer_conditions` / `anti_conditions` 进入 skill contract、scenario families、evaluation cases。
- `context_transfer_abuse` gate，覆盖照搬历史、照搬书本、照搬过去成功经验、照搬竞品/大厂流程。
- 《史记》与 `MaoZeDongAnthology-master` 双样本回归报告。

## 双样本定位

### Shiji

素材形态：

```yaml
narrative_case_heavy: high
history_heavy: high
argument_strategy_heavy: medium
```

重点验证：

- 从叙事案例中抽 `case_mechanism`。
- 从多案例机制 cluster 生成少而精的 judgment skills。
- 拒绝单故事证明、史实查询、人物介绍、古文翻译。

### MaoZeDongAnthology-master

素材形态：

```yaml
argument_strategy_heavy: high
policy_boundary_heavy: high
historical_case_heavy: medium
action_program_heavy: high
```

重点验证：

- 从论述、政策边界、组织路线中抽 `situation_strategy_pattern`。
- 生成现代可用的战略/组织/判断 skills，而不是作者立场问答或政治评论。
- 拒绝纯立场评论、人物介绍、摘要、翻译、历史背景查询。

## 候选 Skill 方向

这些名称是通用使用入口，不绑定具体书名：

- `case-analogy-transfer-gate`
- `mechanism-not-story-mapping`
- `context-transfer-abuse-audit`
- `time-stream-placement`
- `short-term-gain-long-term-cost-check`
- `role-boundary-before-action`
- `principal-contradiction-focus`
- `no-investigation-no-decision`
- `united-front-with-independence`
- `concentrate-superior-force`
- `from-field-feedback-to-policy-loop`
- `policy-boundary-left-right-error-check`

## 验收标准

- 不以书名 hardcode 触发策略；策略由 source shape 触发。
- Graph 层保留摘要、人物、事实、立场、时间线、原文片段等证据，不把“skill 拒绝入口”误解成“不抽取素材”。
- Judgment skills 拒绝纯摘要、翻译、史实查询、人物介绍、立场评论作为触发入口。
- 每条借鉴型 skill 至少绑定 `transfer_conditions` 和 `anti_conditions`。
- 《史记》和 `MaoZeDongAnthology-master` 都能从 raw markdown 独立生成可安装 skills。
- 每个样本至少有 2 条非 gateway judgment skills 通过 release gate。
- 至少一个 benchmark case 覆盖 `single_case_overreach`，至少一个覆盖 `context_transfer_abuse`。
- 质量改善必须来自项目生成逻辑，不允许直接编辑 `/tmp` generated artifacts。

## AGENTS.md 对齐

- External Reference Boundary：Thinking in Time、Analogies at War、cangjie、Graphify 只能作为设计参考或 benchmark，不作为默认生成的隐性输入。
- Workflow-vs-Agentic Boundary：摘要、翻译、事实查询、人物介绍应路由到 retrieval/QA/workflow，不得为追求 rich content 被发布为 thick judgment skill。
- System Efficiency Over Local Advantage：不追求手工挑出的单个优秀历史 skill；以双样本平均生产质量、可迁移性和可审计性为准。

## v0.7 前置关系

v0.6.4 是 v0.7 world-alignment 前的生产质量补强。只有当 biography/history/case-heavy sources 可以稳定产出可迁移 judgment skills 后，world-alignment 才值得叠加；否则外部现实信息会叠在不稳定的 source-to-skill 基座上。
