# 学以致用方法论

学以致用是 KiU 的项目自身方法论。它回答的问题是：如何把原书知识转化为可执行的行动力，同时保留来源证据、使用边界和用户是否引入现实语境的选择权。

## 一句话

```text
原书 -> 读准原书 -> 提炼判断 -> 生成技能 / 分流流程 -> 校准应用 -> 验证行动价值
```

## 1. 读准原书

保留原书事实、段落、锚点、结构和来源。这个步骤可以支持摘要和查询，但它不是最终产品。

失败形态：还没有形成可行动判断，就把原书材料当成行动建议。

## 2. 提炼判断

从原书材料中提炼可迁移判断，回答“这本书支持用户在什么场景下改变判断”。

失败形态：把一本书压缩成流畅文字，但用户仍不知道该如何判断、取舍、拒绝、行动或复盘。

## 3. 生成技能 / 分流流程

能帮助用户判断、取舍、拒绝、行动或复盘的产物，才发布为技能。

确定性强、上下文明确、低判断方差的内容，应分流为流程工件，而不是膨胀成技能。

失败形态：把章节标题、练习题、概念说明、材料容器或确定性流程包装成技能。

## 4. 校准应用

按需加入现实语境、当前事实核验、风险提示和应用门，但不能把外部语境静默混入原书技能。

边界原则：用而不染。现实语境可以改变“是否适用、如何加 caveat、是否拒绝”，但不能改写来源主张。

失败形态：为了更实用而污染原书观点，让用户分不清哪些来自原书、哪些来自现实校准。

## 5. 验证行动价值

验证产物是否在明确证据等级下产生行动价值。行动价值包括发现信号、定义问题、形成判断、选择行动、拒绝误用和复盘校准。

失败形态：把内部审查、脚本评分或同书参照对比说成真实用户验证。

## 与 RAG 的关系

KiU 可以和 RAG、知识库问答、摘要器、摘录库、翻译器放在同一个知识使用语境里比较，但定位不同。

RAG 更偏向从知识库中检索并生成答案；KiU 更关注把原书思想转化为可触发、有边界、能拒绝误用、能指导复盘的行动技能。

## 两个公共表达

- 技能不是摘要：技能必须产生行动力，而不是压缩版章节总结。
- 学到最后要能用：学习的终点不是“知道”，而是能改善具体判断、选择、行动、拒绝和复盘。

## 证据纪律

- 来源忠实、应用安全和行动价值必须分开评分。
- 内部审查不能冒充外部盲评。
- 同书参照对比不能冒充普遍优越性证明。
- 真实用户验证、领域专家审查、外部盲评属于条件依赖型证据，不应被内部指标替代。
