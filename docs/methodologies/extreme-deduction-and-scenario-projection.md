# 极限演绎与场景投影法

极限演绎与场景投影法是 KiU 的项目开发方法论之一。它适用于面对“A 也行、B 也行”的模糊问题，需要快速做结构诊断、命题判断、趋势识别或具体选型。

## 方法定位

极限演绎法：把连续模糊的判断问题，通过极限投影压缩成离散定位问题，再用现实漂移向量定位真相。

场景投影法：在已建好的离散坐标系上，把具体场景投影到对应象限，输出可操作的选型指南。

两者构成双阶段定性方法论：

- L1 极限演绎法立骨架，回答“是什么”。
- L2 场景投影法长肉，回答“该怎么”。

骨没立稳之前，长肉只会长歪。

## L1 极限演绎法

L1 是结构层，目标是把模糊空间离散化成有限结局，再读取现实漂移。

流程：

1. 维度分解：找出 2-7 个关键变量。
2. 极限投影：每个变量推到 0、无穷、正负极端或其他极限状态。
3. 结局塌缩：得到离散单极结局，通常不超过 5 类。
4. 漂移读取：观察现实当下朝哪个结局移动。
5. 定位 / 诊断：落到某单极就给判断；推不出唯一单极，就诊断问题本身可能问错了。

前三步是几何：把模糊空间离散化成坐标系。

后两步是物理：在坐标系里读运动。

## 五类结局

| 结局 | 含义 |
| --- | --- |
| 单极 A | 多数维度向 A 倾斜。 |
| 单极 B | 多数维度向 B 倾斜。 |
| 独立第三方 C | A、B 势均力敌，并且 X 有独立生命周期。 |
| 共享内核 | A、B 在语义上确实共有 X。 |
| 伪命题 | X 不是一个东西，名词糊住了真实结构。 |

伪命题是 L1 最有价值的产物之一。它不是给答案，而是揭示问题本身问错了。

典型伪命题有两种：

- 实体合成：X 实为两个东西被一个名词糊住。例如订单“地址”可能同时指用户地址簿和订单快照。
- 维度合成：争论中的“一根轴”实为两根正交轴。例如 Workflow vs Agent 可能同时包含控制权归属和控制流封闭性。

## L2 场景投影法

L2 是场景层。只有 L1 跑完后仍需具体选型时才启用。

流程：

1. 场景维度化：列出描述真实场景的变量，例如任务结构、容错代价、成本、迭代频率等。注意这是用户面对的世界的变量，不是问题本身的变量。
2. 逐场景投影：把每个场景投到 L1 建好的坐标系上，判断落在哪个象限，并给出权衡曲线。
3. 边界拟合：积累场景后，象限使用边界、决策树和反模式会自然浮现。

L1 交付给 L2 三件事：

- 坐标轴定义：哪几个维度真正关键。
- 离散结局集：答案空间只有这几个，不是无限选项。
- 漂移先验：现实正朝哪个结局移动，用于缩小 L2 搜索。

L2 不能从白板开始，必须在 L1 搭好的坐标系上工作。

## 调用规则

先问：这是结构问题，还是配置问题？

| 问题类型 | 特征 | 用法 |
| --- | --- | --- |
| 结构问题 | 在问 X 究竟是什么、边界在哪、本质是几个东西。 | 用 L1，并停在 L1。 |
| 配置问题 | 结构已清楚，在问具体场景该选哪个配置。 | 先 L1，再 L2。 |
| 混合问题 | 看似配置问题，但反复选不出来。 | 回退 L1，重建坐标系。 |

## 反模式

| 反模式 | 症状 |
| --- | --- |
| 用 L1 产出当 L2 用 | 拿伪命题诊断当选型指南。 |
| 跳过 L1 直接 L2 | 没有坐标系做投影，场景分析变成大海捞针。 |
| L1 维度选错 | 骨架本身错，L2 投得越精细错得越深。 |
| 把骨架清晰当骨架正确 | 方法的整洁感压制怀疑。 |
| 对开放、动态、博弈问题硬套 | 静态快照式方法不适用反身性、博弈、长周期相变问题。 |

## 限制

方法负责优化“在你看见的东西里怎么想”，无法优化“你能看见多少东西”。

方法的天花板就是使用者视野的天花板。这个洞不能在方法内部补，只能由使用者用好奇、谦卑和被惊讶的能力在方法外补。

## 一行总结

L1 立骨：推到极端，看它塌成几种；再看现实在朝哪种漂。漂不出来，就是问题问错了。

L2 长肉：在骨架上把场景投进去，象限的使用边界自然浮现。
