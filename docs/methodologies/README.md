这组文档用于保存 KiU 会调用的方法论工具。`AGENTS.md` 只定义项目调用协议和边界；方法论本体尽量保持通用、可复用，不写成 KiU 专属制度。

## 项目硬边界

项目硬边界保留在 `AGENTS.md`，不属于方法论工具箱。所有方法论调用都必须遵守这些边界，包括来源忠实、证据诚实、外部参考边界、流程/判断边界和应用校准隔离。

## 调用协议

产品和公开叙事优先使用学以致用方法论，避免用历史外部术语解释项目自身。

复杂问题推进优先使用三层递归五步法。

如果问题呈现为“A 也行、B 也行”的结构模糊，再调用极限演绎与场景投影法。

如果问题涉及长期路线、主战场选择或系统方案 vs 局部卓越，再调用系统效率碾压局部优势做一次宏观校准。

## 边界

- 通用思考工具是外挂工具，不是 KiU 产品身份。
- 系统效率原则是提醒式原则，不是机械结论。
- 任何方法论都不能覆盖证据诚实、流程/判断边界、外部参考边界和应用校准隔离。
- 内部方法论可以指导开发，但不能自动升级为真实用户验证或外部评审证据。
