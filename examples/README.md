# Examples

This directory is the minimum regression input set for `v0.6` source/extraction work.

## Sample Books

- `有效需求分析（第2版）.md`
  - engineering/product analysis leaning sample
- `财务报表分析_Markdown版.md`
  - finance/accounting analysis leaning sample

These two markdown books are used as independent source/extraction regression inputs and
are not mixed into a single skill bundle.

## Fixture Bundles

- `fixtures/effective-requirements-analysis.yaml`
- `fixtures/financial-statement-analysis.yaml`

The fixture YAML files remain useful for deterministic pipeline smoke tests, but the two
full markdown books are the canonical inputs for `v0.6` source/extraction validation.
