# v0.7.4 Five-Book Action Identity Rerun Evidence

Date: 2026-04-26
Evidence level: internal deterministic rerun + smoke usage review. This is not external blind review, real-user validation, or domain-expert validation.

## Purpose

v0.7.4 changes generation behavior by adding an `action-skill identity` gate and semantic candidate routing. The rerun checks whether the fix removes source-container pseudo-skills without regressing the established five-book sample set.

The fix is model/routing based:

- Published non-gateway skills must route as `publish_skill` and meet the action identity threshold.
- Thin workflow routers route as `publish_gateway`.
- Deterministic workflow material stays under `workflow_candidates/`.
- Concept notes, case libraries, evaluation material, and source context are preserved under `routed_source_values/` instead of being published as thick skills.
- Heading-like labels that contain action value are normalized into action-skill identities rather than being manually edited in generated artifacts.

## Financial Statement Delta

The important v0.7.4 change is Financial Statement:

- v0.7.3 C-class review found that the old run published section/exercise containers such as `关键概念`, `练习题`, `自主练习`, `分析师工具箱`, `基本练习`, `应用分析`, and `迷你案例` as if they were thick skills.
- v0.7.4 routes those seven items into non-skill audit locations and publishes one thick action skill, `business-value-anchor-check`, plus `workflow-gateway`.
- The release gate remains ready and usage/practical smoke scores remain high, but this is still internal smoke evidence.

## Limits

- This evidence proves internal routing and generated-artifact quality, not real user value.
- Financial Statement breadth is improved relative to fake skill publication, but still not a claim of full financial-analysis domain coverage.
- Proxy usage outputs were not regenerated in this rerun; proxy score fields remain outside this evidence claim.
- The semantic action slug model is intentionally small. Future books may require additional semantic identity families, but they should be added as model-level identity mappings, not sample-specific artifact edits.
