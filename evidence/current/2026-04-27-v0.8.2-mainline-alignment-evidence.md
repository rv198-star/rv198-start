## Structure Baseline

The aligned master tree now uses the v0.8 repository layout:

- `docs/` for public, methodology, engineering, release-history, and archive docs.
- `evidence/` for current and historical evidence.
- `review-pack/current/` for the current reviewer-facing artifact pack.
- `_archive/` for archived imports such as old Notion exports.
- no tracked top-level `reports/`, `notion-export/`, or `workflow_candidates/` directories.

Final structure cleanup added an explicit machine-checkable repository contract:

- tracked root directories must stay inside the intentional v0.8 layout.
- legacy output directories such as `reports/`, `notion-export/`, `workflow_candidates/`, `generated/`, `dist/`, `build/`, `tmp/`, `.worktrees/`, and `.references/` must not be tracked.
- `dist/`, `build/`, and `src/*.egg-info/` must be ignored as local build outputs.
- `_archive/` must carry an explicit boundary index and remain outside the default generation pipeline.
- example fixtures must point at current `examples/source-materials/` paths.
- `review-pack/current/manifest.json` must match the visible review-pack book directories and each book must expose source cards, scorecards, sources, and generated skills.

The contract is implemented in `tests/test_repository_structure.py`.
