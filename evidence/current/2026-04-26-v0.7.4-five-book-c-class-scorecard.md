# v0.7.4 Five-Book C-Class Action-Value Scorecard

Date: 2026-04-26
Evidence level: internal manual C-class review over v0.7.4 rerun artifacts. This is not external blind review, real-user validation, or domain-expert validation.

## Scoring Frame

C-class action value asks whether generated artifacts improve user action under the recursive five-step method:

- `signal_discovery`: world -> signal.
- `problem_definition`: signal -> problem.
- `action_resolution`: problem -> action.
- `feedback_calibration`: execution -> recalibration.

Book-level scoring considers both published thick skills and non-skill routed artifacts. Workflow candidates and source-value routes can receive coverage credit only as routed/auditable material, not as thick skill value.

### User-Facing Dimension Meaning

| Dimension | What a user should read from it |
| --- | --- |
| `source_trust` | Whether the artifact appears faithful to the book and avoids source/world pollution. |
| `action_helpfulness` | Whether the skill helps the user discover signals, define problems, resolve action, or calibrate feedback. |
| `boundary_clarity` | Whether the skill knows when to fire, when to refuse, and when to route to workflow or more context. |
| `coverage_fit` | Whether the book's expected useful value is represented across published skills, gateways, and routed artifacts. |
| `context_application_safety` | Whether world/application context helps use the idea without diluting or rewriting the source. |
| `evidence_confidence` | Whether the claim is internal, proxy, external blind, real-user, or expert-reviewed evidence. |

## Interpretation

v0.7.4 meets the intended optimization target under internal C-class review:

- Financial Statement improves from 72 to 90 by eliminating fake thick skills and publishing two real action-skill identities.
- The other four books all reach 90+ without losing their established published skills.
- `container_candidate_leak_count = 0` for all five reruns.
- `publishable_action_skill_ratio = 1.0000` for all five reruns.

The most important quality change is not a higher traditional usage score. It is that the generated `bundle/skills/` directory now means what it claims: published artifacts are action-skill identities, while source value that is not a skill remains auditable outside the installable skill set.

## Remaining Caveats

- C scores are internal manual judgments and should not be described as real user success.
- Financial Statement still has narrower action-skill breadth than Poor Charlie or Mao. Its v0.7.4 pass means the fake-skill problem is fixed, not that financial-analysis coverage is complete.
- The action identity model will need more semantic families as new domains appear; additions must stay model-level and avoid direct generated-artifact patching.
