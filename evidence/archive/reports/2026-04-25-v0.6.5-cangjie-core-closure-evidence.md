# v0.6.5 Cangjie Core Closure Evidence

Date: 2026-04-25

## Scope

v0.6.5 is an audit-gap closure line for cangjie core absorption. It does not start v0.7 world-alignment and does not use external final `SKILL.md` outputs as hidden generation inputs.

## Implemented In This Batch

- Chapter-title pseudo-skill hard filter before skill promotion.
- `pseudo-skill-audit.json` emitted beside verification reports.
- `generated_bundle.pseudo_skill_audit` surfaced in three-layer review.
- `compatibility_regression` entry surfaced in reference benchmark summaries.
- `ria-tv-stage-report.json` emitted for raw-book runs.
- Extractor responsibility metadata surfaced through the RIA-TV stage report.
- Triple-verification dimensions surfaced in seed verification.
- RIA-TV++ distillation metadata added to generated `candidate.yaml` without rewriting high-quality seeded rationale.
- `final_artifact_effect` remains the two-layer closure gate: usage-only wins cannot close cangjie methodology.
- Decoy pressure packs are generated under `reports/pressure-tests.json` and feed Layer 2 scoring.
- Blind-preference evidence remains explicit and missing-by-default; anonymous A/B evidence can be loaded, but no external blind review was run.
- Same-source cangjie-protocol pressure scenarios were expanded to meet the 20-case release threshold without using external final skill packs as generation inputs.

## Compatibility Notes

- Existing seeded high-quality fixture rationale is preserved verbatim; RIA-TV++ markers are additive metadata unless no explicit rationale exists.
- Workflow-vs-agentic routing remains unchanged: workflow candidates stay outside `bundle/skills/`.
- Graph/source layers may still retain headings, summaries, and factual material; only skill promotion is filtered.

## External Blind Review Pack

A reviewer-facing blind A/B pack is included for v0.6.5 so the remaining external evidence gate can be evaluated without waiting for a new build. The committed pack intentionally excludes the private unblind key.

Committed reviewer files:

- `reports/blind-review-packs/v0.6.5-shiji-cangjie-protocol/reviewer-pack.json`
- `reports/blind-review-packs/v0.6.5-shiji-cangjie-protocol/reviewer-response-template.json`
- `reports/blind-review-packs/v0.6.5-shiji-cangjie-protocol/README.md`

Private key handling:

- Local key generated during release prep: `/tmp/kiu-v065-blind-review-pack-shiji/private-unblind-key.json`
- The key is not committed and must not be sent to reviewers.
- After external reviewers return the filled response template, merge it with `scripts/merge_blind_review_response.py` to produce `blind-preference-review-v0.1` evidence for `scripts/benchmark_reference_pack.py --blind-preference-evidence`.

## Known Limits

- External blind preference review has not been run; the loader rejects non-anonymous labels and keeps missing blind evidence as a hard gate reason.
- Shiji same-source comparison now has 26 protocol scenarios and clears the 20-case release threshold; `same_source_benchmark` no longer blocks the matrix.
- `compatibility_regression.risk` is still reported as `unknown_until_baseline_rerun`; v0.6.4 historical baselines are not replayed by this patch.
- Therefore v0.6.5 can claim `methodology infrastructure ready`, `internal methodology implementation ready`, and `same-source benchmark threshold met`, but not `cangjie methodology absorbed` until external blind evidence is supplied.
