## Finding

The v0.6.3 provenance and multi-file source-coordinate layer now works on `shiji_md`, and the first narrative-history cold-start skill is generated without seed skills or external final SKILL.md inputs.

What changed:

- Multi-file source coordinates are preserved across chunks, graph nodes, graph edges, and scaffolded bundle sources.
- Repeated author boilerplate such as `汉·司马迁` is filtered before candidate extraction.
- A new `narrative_pattern_signal` extractor promotes cross-case decision/consequence evidence into `historical-case-consequence-judgment`.

Current limits:

- The first skill is intentionally broad; it proves the narrative-history path can produce an installable non-gateway skill, but it is not yet a deep multi-skill decomposition of `史记`.
- Usage evidence is still smoke-level (`85.8`), not external real-case review.
- Future work should split richer subskills such as power-transition judgment, incentive/appointment judgment, military-deception judgment, and wealth-production judgment.

## Follow-Up Gate

## Same-Source cangjie Protocol Comparison

The first Shiji smoke result was KiU-only. It proved `raw_book_no_seed_cold_start`, but it did **not** prove a cangjie comparison. Local `.references/cangjie-skill` contains methodology, extractor prompts, and templates, but no executable CLI/runtime and no published same-source Shiji skill pack was found locally.

To avoid misreporting, v0.6.3 now uses an explicit benchmark-only adapter:

- protocol: `cangjie-skill/RIA-TV++-deterministic-adapter`
- official cangjie run: `false`
- input: the same raw `examples/shiji_md/resource` markdown corpus
- output: `/tmp/kiu-v063-shiji-cangjie-protocol/reference-pack`
- generated reference skills: `2`
- generated reference skill ids: `historical-case-consequence-judgment`, `role-boundary-before-action`
- boundary: not KiU default generation, not an external final skill pack input, not a hidden upstream generator

Adapter command:

```bash
PYTHONPATH=src .worktrees/v0.5-foundation/.venv/bin/python scripts/build_cangjie_protocol_baseline.py \
  --input examples/shiji_md/resource \
  --output-root /tmp/kiu-v063-shiji-cangjie-protocol/reference-pack \
  --book-title 史记 \
  --author 司马迁 \
  --source-id shiji-md \
  --publication-year 西汉 \
  --max-chars 1200
```

Benchmark command:

```bash
PYTHONPATH=src .worktrees/v0.5-foundation/.venv/bin/python scripts/benchmark_reference_pack.py \
```

Interpretation:

- KiU wins on structured artifact quality for the one matched skill, mainly because it has KiU-native contract, anchors, evaluation, and revision structures.
- KiU loses the same-scenario usage comparison against the cangjie protocol baseline by `8.1` points.
- KiU generated only one Shiji skill, while the cangjie protocol baseline generated two; `role-boundary-before-action` is unmatched by KiU.
- The main KiU failure tags are `generic_reasoning`, `boundary_leak`, and `next_step_blunt`.
- Dominant repair targets are `evidence_summary.anchor_refs`, `usage_summary.representative_cases`, `contract.boundary.do_not_fire_when`, and `contract.boundary.fails_when`.

Revised v0.6.3 conclusion:

- Cold-start provenance and multi-file source-coordinate support are proven.
- The Shiji narrative-history path is installable but not yet competitive in usage quality.
- cangjie methodology absorption remains incomplete; the next work should improve cold-start decomposition, boundary probes, and scenario-grounded usage language rather than claim victory from KiU-only smoke scores.
