### Reading The Scores

- `book_to_skill_action_coverage_100` asks whether the book's expected action value is covered by generated skills plus workflow candidates. This is different from whether the published skills are individually good.
- `skill_action_value_average_100` averages the published `SKILL.md` artifacts only. It does not credit workflow candidates unless a gateway makes them usable.
- `action_value_score_100` is the final internal C-class score for the generated artifacts in this run.
- `财务报表分析` is intentionally scored much lower than its traditional proxy/practical scores because several published skill ids are section/exercise-shaped (`自主练习`, `练习题`, `关键概念`, etc.). This is exactly the kind of mismatch C-class scoring is meant to reveal.

## Book-Level Findings

### `有效需求分析（第2版）`

This run has good C-class structure. It converts the book into two thick judgment skills plus a workflow gateway and preserves twelve workflow candidates. The main gap is that workflow-action-value is only marked `present_but_not_scored`, so book-level coverage is lower than the individual skill scores.

### `财务报表分析`

This is the clearest case where old scores and C-class scores diverge. Traditional usage/proxy/practical scores are high, but C-class action value is only `72` because many published skills look like section or exercise containers rather than installable judgment contracts. C-class scoring therefore flags a real book-to-skill conversion weakness that old Usage did not expose.

### `毛泽东选集 / Anthology`

This run has strong action value because `no-investigation-no-decision` and `principal-contradiction-focus` are directly usable mental models. The historical/role-boundary skills add transfer safety. Book-level coverage remains below Poor Charlie because the anthology is broad and still has workflow/gateway coverage rather than full action-value closure.

### `Poor Charlie's Almanack`

This remains the strongest book-to-skill action-value run. The skill set is coherent, mature, and well matched to the book's mental-model nature. Its main limitation is evidence level, not artifact structure: it remains internal proxy/manual review evidence.

### `史记`

The three generated skills have strong individual action value for historical borrowing, but book-level coverage is only `82` because `史记` is extremely broad and the run intentionally extracts cross-chapter judgment skills rather than covering the whole book. This is acceptable for borrowed-value cold start, but not full book coverage.

## Conclusion

The five-book C-class view changes the interpretation of v0.7.2 readiness evidence:

- High traditional Usage/Proxy scores do not necessarily mean high book-to-skill action value.
- Poor Charlie, Mao, and Shiji show strong generated judgment skills.
- Effective Requirements shows good split between agentic skills and workflow candidates.
- Financial Statement exposes a real conversion weakness: high practical proxy but weak published skill identities.

This is the view v0.7.3 should use when judging whether generated `SKILL.md` artifacts create usable action value.
