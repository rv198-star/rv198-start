# v0.7.0 Basic Isolated World Alignment Implementation Evidence

This is an implementation and effect-check report, not a release report. v0.7.0 is not published from this run.

## Scope

Implemented the v0.7.0 basic-structure version of `学以致用 · World Alignment`:

- `world_context.yaml` as the isolated world-context layer.
- `application_gate.yaml` as the only application verdict outlet.
- `WORLD_ALIGNMENT.md` as the human/LLM-readable explanation layer.
- No-web mode as the default for v0.7.0 construction.
- Deterministic keyword preflight plus agentic-principle no-web review as two-layer hallucination control.
- Workflow-plus-agentic-proxy deepening loop for world context: deterministic workflow derives candidate pressure dimensions, then an agentic-style critique/revise loop rejects generic context and no-web hallucination risk.
- Generated-run review now includes a `world_alignment` scorecard when artifacts are present.
- Deterministic randomized proxy-usage review as a downgrade path when human user testing or external blind review is unavailable.

Explicitly not implemented in v0.7.0:

- Live web search.
- Real-time fact verification.
- Multi-world / multi-perspective / multi-stance modeling.
- `valid_until` lifecycle management.
- Automatic current financial or investment-data checks.

## No-Web Risk Control

The no-web gate has two layers:

1. Deterministic keyword/structure preflight catches explicit current-fact claims such as `最新`, `当前市场`, `截至今天`, and high-sensitivity direct-apply attempts.
2. Agentic-principle review catches semantic risk patterns such as unverified claims stated as facts or overgeneralized claims like `彻底改变所有`.

The tests include adversarial no-web text and require both layers to detect it.

## World-Context Depth Gate

The initial v0.7.0 implementation proved only structure and isolation. It produced generic context such as "real-world organizational practices and tooling may change how this method should be applied." That was insufficient: it did not model a concrete reality pressure, did not distinguish workflow parsing from agentic judgment, and could still receive a perfect structure score.

The current implementation adds a depth loop:

1. Workflow step: parse skill metadata, classify temporal sensitivity, and derive candidate pressure dimensions.
2. Agentic proxy step: select the most relevant pressure dimensions, draft a world hypothesis, and ask what user context changes safe application.
3. Critique step: reject generic phrases, missing domain pressure, unverified current-fact claims, or source-rewrite risk.
4. Revision step: deepen the pressure set until the item passes or the loop cap is reached.

Artifacts now expose `pressure_dimensions`, `world_hypothesis`, `why_this_matters`, `user_context_questions`, `loop_mode`, `deepening_rounds`, `depth_notes`, and `world_context_depth_score`. Review scoring now includes `world_context_depth_score`, so a generic world context can pull `world_alignment_score_100` below the release threshold even when isolation passes.

## Proxy Usage Downgrade Evaluation

Because real user testing and external blind review are not currently available, v0.7.0 adds a lower-confidence proxy usage layer. This does not replace `usage_outputs` and does not change the three-layer overall score. It is reported separately as `proxy_usage_outputs`.

The proxy generator creates deterministic randomized user-like prompts from skill semantics and world-alignment pressure dimensions. It covers five case types:

- `should_fire`
- `should_not_fire`
- `edge_case`
- `world_alignment_case`
- `high_risk_or_missing_context`

The proxy evaluator predicts verdicts from prompt and artifact signals, not directly from expected verdict or `case_type`. Its claim boundary is explicit: proxy usage is an internal regression guard, not real user validation or external blind review.

Current proxy results:

| --- | ---: | ---: | ---: | --- | ---: | --- | --- |
| Effective Requirements | 95.8 | 100.0 | 95.2 | `L2_5_proxy_usage` | 10 | `{}` | ready |
| Financial Statement | 95.8 | 96.2 | 95.8 | `L2_5_proxy_usage` | 20 | `{'generic_prompt': 5}` | ready |

Interpretation: the proxy layer gives a better guardrail than smoke usage because it tests varied prompts such as translation requests, concept queries, boundary-unclear cases, world-alignment pressure, and direct high-risk advice requests. The `practical_effect_outputs` score blends existing smoke usage with capped proxy usage, so proxy cannot inflate a run to 100. It remains weaker than L3 human/external blind review and should not be used to claim real-world effectiveness.

The v0.7.0 basic-version usage target is therefore met in the generated usage-regression sense: both current raw-book runs report `usage_outputs.score_100 = 95.8`. This does not close the later L3/L4 effectiveness backlog, and it does not replace historical cangjie same-source comparison. It only says the current v0.7 artifacts are no worse than the v0.6.x generated-usage baseline under KiU's internal three-layer review.


## Need-Score And Relevance-Arbitration Correction

After reviewing the first v0.7.0 world-alignment outputs, we found that low-temporal principle skills could receive off-domain modern pressures. The clearest failure mode was `circle-of-competence` being enriched with business / AI-prototyping / subsystem-boundary language. That was safe from a no-web perspective, but it diluted the source mental model.

The correction adds four release-blocking capabilities:

- `world_alignment_need_score` with `minimal`, `light`, `moderate`, and `strong_gate` intervention levels.
- Candidate pressure relevance arbitration with `source_fit_score`, `enrichment_value_score`, `application_need_score`, `dilution_risk_score`, and `hallucination_risk_score`.
- `no_forced_enhancement` support when no candidate pressure improves the source-derived skill.
- Review penalties for weak source fit, source dilution, and unverified current-fact hallucination even when the context is structurally detailed.

Additional hallucination tests now cover English no-web current-fact claims such as `Latest market data proves this company is undervalued today`, not only Chinese `最新/当前` patterns. In no-web mode these claims fail risk control and lower the world-alignment score.

Post-correction sample review:

| Sample | World alignment | Source fit | Dilution risk | Hallucination risk | Accepted pressures | Rejected pressures | No forced enhancement | Verdicts |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: | --- |
| Effective Requirements | 97.0 | 85.0 | 10.0 | 10.0 | 3 | 13 | 0 | `apply_with_caveats: 1` |
| Financial Statement | 97.0 | 85.0 | 10.0 | 10.0 | 6 | 12 | 0 | `ask_more_context: 2` |
| Poor Charlie | 94.6 | 83.5 | 10.0 | 10.0 | 8 | 62 | 0 | `apply: 5`, `ask_more_context: 1` |

The Poor Charlie correction is the key evidence. `circle-of-competence` now has:

- `world_alignment_need_score = 0.195`
- `intervention_level = minimal`
- accepted pressure: `expertise boundaries can shift when tools expand surface competence...`
- rejected off-domain pressure: AI prototyping / subsystem-boundary / business-accountability style candidates

This is not a claim that low-temporal principle modeling is perfect. It is a narrower claim: v0.7.0 no longer rewards detailed but source-diluting world context, and it can keep low-need principle skills in a light-touch mode.

## Current Limitations

- The world-context generator is no longer purely generic, but it remains a deterministic agentic proxy. It improves specificity and application usefulness; it does not prove factual correctness about the current world.
- Agentic-principle review is implemented as deterministic principle heuristics in this pass, not an external LLM reviewer.
- `refuse` is currently covered by a targeted adversarial fixture rather than a real book sample. Before release, decide whether that is sufficient or whether a real high-risk sample is required.
- Some scores can still be high because the scorecard checks artifact quality and no-web safety, not external truth. External factual validation and live web checks remain future work.
- Proxy usage prompts are still generated by KiU itself. They reduce smoke-test shallowness but can miss real user phrasing, adversarial creativity, and domain expert objections.
- No release should be cut from this implementation pass without a follow-up review and at least one targeted refuse/adversarial evidence case.
