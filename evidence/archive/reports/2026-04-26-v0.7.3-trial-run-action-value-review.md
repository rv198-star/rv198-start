## Book-to-Skill Action Value Map

### Book Action-Value Expectation

`有效需求分析（第2版）` is a requirements-analysis method book. Its expected action value is not primarily concept explanation. A successful KiU run should help users:

- discover requirement signals hidden behind customer feature requests, stakeholder resistance, process symptoms, and missing users;
- define surface requests as falsifiable business or user problems before accepting a solution;
- route deterministic analysis work into executable workflows instead of pretending every checklist is an agentic skill;
- resolve judgment-heavy cases into decisions such as reframe, accept with constraints, ask for missing stakeholder, defer, or route to workflow;
- preserve a boundary between workflow execution and agentic judgment.

## What the New Taxonomy Explains Better

The old review says `Usage = 97.9` and `Overall = 94.4`. That is useful but under-explains the artifact value.

The A/B/C taxonomy explains:

- Why `workflow-gateway` is not a weak thick skill: its value is boundary routing, not domain judgment.
- Why `solution-to-problem-reframing` is valuable even without direct implementation steps: its primary value is L2 problem definition.
- Why workflow-heavy content should not be penalized for being routed out of `bundle/skills/`: workflow routing preserves the workflow-vs-agentic boundary.
- Why missing proxy/world outputs should be reported as missing evidence, not silently converted into quality claims.

## Remaining Gaps

- This trial did not generate proxy usage cases, so C-class scores are manual internal review rather than automated proxy evidence.
- No same-book reference comparison was run, so `comparative_decision_lift_100` is qualitative.
- Feedback calibration is present as deferral and missing-context handling, but not as decision logs, pre-mortems, or hit-rate ledgers.
- The run is one book and should not be used to claim cross-domain action value.

## Trial Conclusion

The v0.7.3 taxonomy is useful. It separates:

- Class A: the run follows KiU method and workflow-agentic boundaries;
- Class B: source provenance and pollution safety are intact;
- Class C: the main user value is problem definition and boundary-aware routing, not generic action-list generation.

This supports the v0.7.3 direction: adjust evaluation vocabulary and evidence reporting before changing generation behavior.
