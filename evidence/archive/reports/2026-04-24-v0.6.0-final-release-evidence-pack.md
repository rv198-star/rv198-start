# KiU v0.6.0 Final Release Evidence Pack

Date: `2026-04-24`

## Release Judgment

`v0.6.0` can be treated as the source/provenance release line for KiU. The release claim is intentionally bounded:

- It does close the `In Use + provenance + workflow-boundary` line.
- It does close `Graphify` core absorption on the release benchmark; it does not claim parity with Graphify peripheral tooling such as IDE integrations, MCP serving, or export scaffolding.
- It does not claim `v0.7`-style world-alignment.
- It does not use external reference packs as hidden upstream generators.

The release remains aligned with `AGENTS.md`: external projects are benchmark/reference material only, and workflow-vs-agentic boundary quality is a hard standard.

Version positioning note:

`v0.6.0` should be read as a bottom-layer closure, not as proof that final SKILL usage already massively exceeds `cangjie-skill`. The release proves that KiU now has a stronger provenance/tri-state/graph substrate while preserving a modest same-source usage lead. The next quality frontier is graph-to-skill distillation: turning that substrate into sharper triggers, boundary cases, and action language.

## KIU-650 Boundary Cleanup

The final boundary cleanup targets concept-query leakage in `invert-the-problem`. The source bundle now explicitly rejects:

- pure concept definitions
- method-history questions
- historical-example lookup prompts

This is a boundary-preserving fix. It does not broaden inversion to win benchmark points; it narrows firing conditions so explanatory prompts do not become fake live decisions.
