# v0.5.1 Scoring Paradox Reverse Diagnosis

Date: 2026-04-25

## Question

Why could `v0.5.1` slightly beat the local `cangjie-skill` reference in same-scenario usage while `cangjie_core_absorbed_100` stayed low?

## Finding

The result is explainable, but the old evaluation was incomplete.

`v0.5.1` mostly won on KiU-native strengths:

- explicit workflow-vs-agentic boundary
- double-anchor evidence discipline
- refusal / boundary handling
- clearer next-action language
- the repaired `value-assessment` parent skill topology

Those are real quality gains, but they are not the same thing as absorbing cangjie's book-to-skill production methodology.

## Old Rubric Bias

The previous reference benchmark tracked `cangjie_core_absorbed_100` with mostly observable engineering proxies:

- pipeline/stage artifact presence
- extractor kind coverage
- generated output count vs reference count
- same-scenario usage score

This made it possible for KiU to win usage while still not proving:

- RIA-TV++ stage separation
- deep principle extraction
- cross-chapter synthesis
- triple verification
- decoy/adversarial pressure testing
- blind preference under equal source conditions

So the old conclusion should be read narrowly: `v0.5.1` closed the same-source/same-scenario usage gap. It did not close the cangjie methodology gap.

## New Acceptance Split

The benchmark now separates two claims:

- `cangjie_core_absorbed_100`: structural/reference absorption proxy.
- `cangjie_methodology_quality_100`: production-methodology proof.
- `final_artifact_effect`: two-layer final output effect gate.

A run may have high `cangjie_core_absorbed_100` and still fail `cangjie_methodology_gate`.

If same-scenario usage wins but methodology proof is missing, the gate reports:

- `ready = false`
- `claim = same_scenario_usage_win_only`

At the final output level this becomes:

- `final_artifact_effect.claim = usage_effect_only`
- not `two_layer_effect_proven`

This distinction matters because both layers are final-output effects. Layer 1 checks
whether a user can use the skill immediately and safely. Layer 2 checks whether the
skill contains deep, transferable, pressure-tested knowledge rather than only clean
usage packaging.

## Required Evidence Before Claiming Methodology Closure

Future cangjie methodology closure requires evidence for all of these:

- `principle_depth_review`
- `cross_chapter_synthesis`
- `triple_verification`
- `decoy_pressure_test`
- `blind_preference_review`

Minimum gate:

- `cangjie_methodology_quality_100 >= 80.0`
- each core methodology evidence ratio >= `0.8`
- same-scenario usage does not regress
- `final_artifact_effect.claim = two_layer_effect_proven`

## Implication

The evaluation system was not useless, but it was under-scoped. It measured usage hygiene and KiU's engineering strengths better than it measured cangjie's deep production method. The next v0.6.x work should first raise this methodology gate rather than declaring cangjie fully absorbed from usage delta alone.
